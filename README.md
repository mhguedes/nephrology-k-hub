# Nephrology K-Grant Hub

A living, weekly-updated web hub that centralizes NIH career-development (**K**) awards in
kidney research — a grantsmanship resource for nephrology trainees and faculty.

It delivers three things:

1. **A grant library** of funded nephrology K awards, organized by subarea (CKD, ESKD/dialysis,
   transplant, glomerular disease, AKI, hypertension, PKD/genetic, stones/electrolytes,
   pediatric, health services/epi) and split into **clinical** (trials / observational) vs.
   **non-clinical / basic** research.
2. **An institutional map** of US universities ranked by number of active nephrology K awards
   (with the clinical / non-clinical split), plus a state-level geographic view.
3. **Weekly auto-refresh** from NIH RePORTER.

## Viewing it

Open `index.html` in a browser. On a double-click (file://) it shows an embedded real sample
of awards plus the full June 2026 snapshot statistics. When served over HTTP (so it can read
`data.json`), it shows the complete current dataset:

```bash
cd nephrology-k-hub
python -m http.server 8000
# visit http://localhost:8000
```

## Refreshing the data

The page can't call the RePORTER API directly (the API blocks in-browser cross-origin requests),
so data is pulled **server-side** by the pipeline:

```bash
python pipeline/reporter_pull.py                 # high-precision filter, last 3 FYs -> ../data.json
python pipeline/reporter_pull.py --loose         # broader superset (incl. onco-nephrology/urology)
python pipeline/reporter_pull.py --years 2020 2021 2022 2023 2024 2025 2026   # wider history
```

### Full record / proposal links

Each award links to its **NIH RePORTER record** (complete public abstract, public-health-relevance
statement, investigators, study section, funding) and, where available, its funding-opportunity
(FOA) number. **NIH does not publish the full grant application** (Specific Aims / Research
Strategy) — those are FOIA-only — so there is no public proposal PDF; the RePORTER record is the
authoritative public document. A future enhancement will attach each award's **publications**
(RePORTER publications endpoint → PubMed/PMC) as the closest proxy for the funded work.

Run it anywhere with normal internet access (your laptop, a server, or CI). The included
GitHub Actions workflow runs it every Monday and commits the refreshed `data.json`; pair it with
GitHub Pages (or any static host) to publish a self-updating site.

## How awards are classified

- **Scope:** active K-series mechanisms (K01, K08, K23, K99, K24, K25, K38, K43, K02…) funded by
  NIDDK and other ICs (NHLBI, NIA, NIAID, NICHD…). An award is included when **kidney disease is the
  study focus or study population** — a kidney-disease term in the title, *or* an explicit
  kidney-disease cohort named in the abstract ("patients on dialysis", "kidney transplant
  recipients", "adults with CKD"…). This keeps **comorbidity studies that enroll kidney-disease
  patients** (anemia, cardiovascular, metabolic studies in CKD/dialysis/transplant cohorts) and
  excludes incidental kidney mentions and pure kidney-cancer/urology (~159 awards as of June 2026).
  Add `--loose` for a maximum-recall abstract-level match.
- **Subarea:** keyword scoring against the standard clinical taxonomy.
- **Clinical vs. non-clinical:** mechanism prior + clinical-trial/cohort language vs.
  basic-science language; clinical is further split into trial vs. observational.

These are **first-pass algorithmic labels for triage and contain errors.** The schema keeps an
editable label so a human can curate.

## Known limitations / roadmap

- **Mentors (Goal 2b):** RePORTER does not publish K mentor/sponsor names. A mentor ranking can't
  come straight from the data. Planned: infer likely mentors from senior/corresponding authors on
  each scholar's K-supported publications (RePORTER `publications` endpoint → PubMed), labeled as
  inferred, with optional manual curation.
- **K-equivalent awards (VA CDA, DoD/CDMRP, foundation career awards):** not in RePORTER's clean
  API; would be a manually curated Phase 2 supplement.
- **Publication linkage:** stubbed — add a call to `/v2/publications/search` keyed on
  `core_project_num` to attach each award's papers.
- **Classification accuracy:** validate a sample and refine keyword dictionaries; consider an
  embedding/LLM classifier for the clinical vs. non-clinical and subarea labels.

## Data source

NIH RePORTER API v2 — <https://api.reporter.nih.gov>. Public domain U.S. government data.
This is a research/triage tool, not an official NIH product.
