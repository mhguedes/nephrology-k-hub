How this hub is built

Source. All data come from the NIH RePORTER database (API v2), the authoritative public record of NIH-funded awards. NIH refreshes RePORTER weekly.

Scope. Active career-development (K) awards — K01, K08, K23, K99, K24, K25, K38, K43, K02 and related — funded by NIDDK and other institutes (NHLBI, NIA, NIAID, NICHD, and others). An award is included when kidney disease is the study focus or the study population: a kidney-disease term appears in the title, or the abstract names an explicit kidney-disease population ("patients on dialysis", "kidney transplant recipients", "adults with CKD", etc.). This deliberately keeps comorbidity studies that enroll kidney-disease patients (e.g., anemia, cardiovascular, or metabolic studies in CKD/dialysis/transplant cohorts), while excluding studies that only mention the kidney incidentally and pure kidney-cancer/urology. The pipeline also offers a looser abstract-level mode for maximum recall.

Full record & proposal documents

Each award links to its NIH RePORTER record, which contains the complete public abstract, public-health-relevance statement, investigators, study section, project period, and funding. NIH does not publish the full grant application (Specific Aims, Research Strategy) — those are available only via FOIA — so there is no public proposal PDF to link. Where a funding opportunity (FOA/NOFO) number is available it is shown and linked. For funded science, the award's linked publications (a planned addition via the RePORTER publications endpoint → PubMed/PMC) are the closest thing to the proposal's results.

Clinical vs. non-clinical (Goal 1.1)

Each award is labeled by combining (1) a mechanism prior (K23/K24 lean patient-oriented; K08/K99/K01 lean basic), (2) clinical-trial / cohort language in the abstract, and (3) basic-science language (animal models, in vitro, mechanism). Clinical awards are further split into clinical trial vs. observational. These are first-pass algorithmic labels for triage and contain errors; a human-reviewed override is intended.

Institutional map (Goal 2)

Built by aggregating the administering organization and state across all matched awards, with the clinical/non-clinical split preserved.

Mentors & co-mentors (Goal 2) — known limitation

RePORTER exposes the K scholar (awardee) and NIH program officers, but not the mentor or sponsor. A reliable mentor ranking therefore cannot come straight from the data. The planned approach is to infer likely mentors from the senior/corresponding authors on each scholar's K-supported publications, then aggregate — clearly labeled as inferred — with optional manual curation.

Dynamic weekly updates (Goal 3)

The page reads data.json. A scheduled job (pipeline/reporter_pull.py, runnable weekly via the included GitHub Actions workflow) re-queries RePORTER, re-classifies, recomputes the aggregates, and rewrites data.json. Because the RePORTER API blocks direct in-browser calls (no CORS), the fetch must run server-side in the pipeline rather than from this page.

Caveats

The text filter over-recalls (some non-kidney NIDDK/NCI/NIAID awards match generic terms like "transplant"); the snapshot here applies a tighter nephrology gate. Counts are a point-in-time snapshot of active awards, not a complete historical record — the pipeline can be configured to include past fiscal years.
